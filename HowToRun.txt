===========================
HOW TO RUN THE PROJECT
===========================

PREREQUISITES
-------------
Make sure the following are installed on your machine before starting:

  - Node.js  v18 or later   -> https://nodejs.org
  - Python   v3.11 or later -> https://www.python.org
  - Git                     -> https://git-scm.com

---

STEP 1 — Clone and checkout the branch
---------------------------------------
  git clone https://github.com/Kyanqueso/admin_fullstack.git
  cd admin_fullstack
  git checkout mostly_complete

---

STEP 2 — Set up the backend (Python / FastAPI)
------------------------------------------------
Open a terminal in the project root and run:

  cd api

  # Create a virtual environment
  python -m venv venv

  # Activate it
  # Windows (CMD / PowerShell):
  .\venv\Scripts\activate

  # Mac / Linux:
  source venv/bin/activate

  # Install dependencies
  pip install -r requirements.txt

Verify that api/.env exists and contains the correct values.
It should look like this:

  SUPABASE_URL=...
  SUPABASE_KEY=...
  DATABASE_URL=...
  SUPABASE_JWT_SECRET=...

(Obtain these values from the project owner if not included in the branch.)

---

STEP 3 — Set up the frontend (Node.js / Electron)
---------------------------------------------------
Open a new terminal in the project root and run:

  cd web
  npm install

Verify that web/.env exists and contains the correct values.
It should look like this:

  VITE_BACKEND_URL=http://localhost:8000
  VITE_SUPABASE_URL=...
  VITE_SUPABASE_KEY=...

---

STEP 4 — Build the frontend
-----------------------------
Still inside the web/ folder, build the Vite bundle:

  npm run build

This generates the web/dist/ folder that Electron loads.
You must re-run this step any time you change frontend source files.

---

STEP 5 — Run the application
------------------------------
Still inside the web/ folder:

  npm start

Backend must be activated!!!

---

NOTES
-----
- Run steps 2–5 in order on first setup. After that, only steps 4–5 are needed
  for regular use (skip step 4 if you haven't changed any frontend files).

- Do NOT commit the following folders — they are in .gitignore:
    web/node_modules/
    api/venv/

- The web/dist/ folder is committed to the repo so Electron can load it.
  If dist/ is missing or outdated, always run: npm run build

---

TROUBLESHOOTING
---------------

If the app opens but the backend does not connect (API errors in the UI):

  1. Open a terminal and manually check if uvicorn starts correctly:
       cd api
       .\venv\Scripts\activate   (Windows) / source venv/bin/activate (Mac/Linux)
       uvicorn app.main:app --reload

  2. If you see import errors or missing packages, reinstall:
       pip install --upgrade --force-reinstall -r requirements.txt

  3. Confirm api/.env is present and has valid values.

If npm start fails with a missing module error:

  cd web
  npm install
  npm run build
  npm start
